# Practice Tasks: Module 6 - Terraform IaC

This document outlines step-by-step tasks to practice Infrastructure as Code (IaC) configuration, state locking, and resource importing.

---

## Task 1: Basic Resource Provisioning
*   **Goal**: Create a Terraform configuration file to declare and provision a local file and directory path.
*   **Step-by-Step Instructions**:
    1. Create a workspace directory:
       ```bash
       mkdir -p /tmp/tf-lab
       cd /tmp/tf-lab
       ```
    2. Create a configuration file `main.tf`:
       ```hcl
       # /tmp/tf-lab/main.tf
       terraform {
         required_providers {
           local = {
             source  = "hashicorp/local"
             version = "~> 2.0"
           }
         }
       }

       resource "local_file" "config" {
         content  = "database_host = 'localhost'\nport = 5432\n"
         filename = "${path.module}/app_config.txt"
       }
       ```
       Write this file to disk:
       ```bash
       tee /tmp/tf-lab/main.tf << 'EOF'
       terraform {
         required_providers {
           local = {
             source  = "hashicorp/local"
             version = "~> 2.0"
           }
         }
       }

       resource "local_file" "config" {
         content  = "database_host = 'localhost'\nport = 5432\n"
         filename = "${path.module}/app_config.txt"
       }
       EOF
       ```
    3. Initialize the working directory:
       ```bash
       terraform init
       ```
    4. Plan the resource creation:
       ```bash
       terraform plan -out=tfplan
       ```
    5. Apply the plan to create the file:
       ```bash
       terraform apply tfplan
       ```
*   **Verification**:
    Verify the generated configuration file content:
    ```bash
    cat /tmp/tf-lab/app_config.txt
    ```

## Task 2: Parameter Overrides using Variables
*   **Goal**: Define inputs and variables to customize resource output values dynamically.
*   **Step-by-Step Instructions**:
    1. Create a `variables.tf` file:
       ```hcl
       # /tmp/tf-lab/variables.tf
       variable "db_port" {
         type    = number
         default = 5432
       }

       variable "db_host" {
         type    = string
         default = "127.0.0.1"
       }
       ```
       Write this file to disk:
       ```bash
       tee /tmp/tf-lab/variables.tf << 'EOF'
       variable "db_port" {
         type    = number
         default = 5432
       }

       variable "db_host" {
         type    = string
         default = "127.0.0.1"
       }
       EOF
       ```
    2. Modify the file resource configuration in `main.tf` to parse these variables:
       ```hcl
       # Update in /tmp/tf-lab/main.tf
       resource "local_file" "config" {
         content  = "database_host = '${var.db_host}'\nport = ${var.db_port}\n"
         filename = "${path.module}/app_config.txt"
       }
       ```
       Replace `main.tf` contents:
       ```bash
       tee /tmp/tf-lab/main.tf << 'EOF'
       terraform {
         required_providers {
           local = {
             source  = "hashicorp/local"
             version = "~> 2.0"
           }
         }
       }

       resource "local_file" "config" {
         content  = "database_host = '${var.db_host}'\nport = ${var.db_port}\n"
         filename = "${path.module}/app_config.txt"
       }
       EOF
       ```
    3. Apply changes using custom inputs via `--var` parameters:
       ```bash
       terraform apply -var="db_port=6432" -var="db_host=postgres.internal" -auto-approve
       ```
*   **Verification**:
    Verify the file contains the overridden values:
    ```bash
    cat /tmp/tf-lab/app_config.txt
    ```

## Task 3: Drift Detection and Rectification
*   **Goal**: Manually modify a managed resource and run drift checks to identify differences.
*   **Step-by-Step Instructions**:
    1. Manually modify the file generated by Terraform:
       ```bash
       echo "database_host = 'hacked_host'" > /tmp/tf-lab/app_config.txt
       ```
    2. Run the plan command. Terraform should detect the drift:
       ```bash
       terraform plan
       ```
    3. Run apply to overwrite the manual modifications and restore the desired configuration:
       ```bash
       terraform apply -auto-approve
       ```
*   **Verification**:
    Verify the file has been restored to match the state configuration settings:
    ```bash
    cat /tmp/tf-lab/app_config.txt
    ```
    Clean up resources:
    ```bash
    terraform destroy -auto-approve
    ```
