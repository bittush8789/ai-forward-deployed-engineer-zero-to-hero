# System Design for Enterprise AI Platforms (Phase 15)

Welcome to the **System Design** curriculum. This curriculum focuses on distributed systems, scalability patterns, high availability layouts, disaster recovery strategies, event-driven architectures, enterprise systems design, AI platforms topologies, and multi-tenant architectures.

### Modules
| # | Module | Description |
|---|--------|-------------|
| 01 | [Distributed Systems](./modules/01_distributed_systems.md) | REST/gRPC service communication, data replication/sharding, CAP theorem, and Saga/CQRS design patterns. |
| 02 | [Scalability](./modules/02_scalability.md) | Horizontal/vertical scaling, load balancers, database sharding, and queue-based scaling. |
| 03 | [High Availability](./modules/03_high_availability.md) | Active-Active/Active-Passive setups, primary-secondary database replication, and Multi-AZ layouts. |
| 04 | [Disaster Recovery](./modules/04_disaster_recovery.md) | RTO/RPO objectives, Velero backups, cross-region replication, and failover checks. |
| 05 | [Event-Driven Systems](./modules/05_event_driven.md) | Pub/Sub, Kafka event streams, consumer decoupling, and event pipelines. |
| 06 | [Enterprise Architecture](./modules/06_enterprise_architecture.md) | Microservices, Domain-Driven Design (DDD), and hexagonal architectures. |
| 07 | [AI System Design](./modules/07_ai_system_design.md) | Data, training, serving, and monitoring layers in production AI platforms. |
| 08 | [Multi-Tenant Architecture](./modules/08_multi_tenant.md) | Shared DB vs separate schemas, computing bounds, Keycloak authentication, and usage metering. |
| 09 | [Capstone Projects](./modules/09_capstone_projects.md) | Five capstone systems design projects covering RAG platforms, multi-agent routing, Triton inference, MLOps, and SaaS portals. |
